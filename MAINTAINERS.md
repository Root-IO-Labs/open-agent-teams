# Maintainers

This document lists the current maintainers of Open Agent Teams (OAT) and the
areas each person owns. Maintainers are expected to review pull requests,
triage issues, and shepherd releases for their areas.

## Becoming a Maintainer

The current maintainers nominate new maintainers based on sustained, high-quality
contributions to the project. There is no formal application process — if you
are interested in taking on more responsibility, the fastest path is to submit
pull requests, review others' pull requests, and participate in issue triage.

## Maintainer Responsibilities

- Respond to issues and pull requests in your areas within a reasonable
  timeframe (best-effort; this is an OSS project).
- Uphold the [Code of Conduct](CODE_OF_CONDUCT.md).
- Follow [Contributing guidelines](CONTRIBUTING.md) and hold contributions to the same bar.
- Help shepherd releases by triaging CHANGELOG entries, testing release candidates,
  and coordinating the tag + `goreleaser` flow described in
  [`docs/RELEASING.md`](docs/RELEASING.md) when it exists (WIP).
